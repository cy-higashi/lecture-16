03 育成方針

共通言語として、
まず何かをつくってみましょう

なぜ「つくる体験」が必要か
  • エンジニアになることが目的ではない
  • お互いが何を話しているか理解できる「共通言語」を持つことが目的
  • 「つくれる」より「つくろうとしたことがある」が重要
  • 技術の限界・可能性を自分の言葉で語れるようになる
何をつくるか
  • 今自分がやっている業務の中で「これ自動化できそう」を一つ選ぶ
  • 完成度より「動く」を体験することに意味がある
  • 一つ動かすと、次の打ち手の解像度が格段に上がる
  • ここから先の学習スピードが根本的に変わる

提案の本質： 技術を習得させるのではなく、「自分のアイデアをシステムとして表現できる状態」を作ること。
それがあなたのキャリア全体の土台になる。

03 育成方針 — アプローチの3レイヤー

考え方の3レイヤー

01THINK

フローチャートの考え方を常に頭に入れておく

業務・アイデア・課題を「フロー」として整理する習慣を持つ。
どんなツールを使う前にも、まず構造として整理できているかが全ての起点になる。
粒度さえ間違えなければ正しいものは必ずできる。AIが伴走すれば即作れる。

02BUILD

ノーコード・ローコードによる打ち手を柔軟に持つ

GAS・Make・Notion APIなど、コードを最小限に動く仕組みを組む手段を持つ。
課題と手段を素早くマッチングできる判断力が最大の武器になる。
「何を使えば最短で解決するか」を選べる人間になることがゴール。

03SCALE

AIエージェントを活用したよりテック的なアプローチ

Dify・LLM API連携など、より高度な自動化・知識構造化に踏み込む。
「使える人間」から「設計できる人間」へのステップ。
ここに至れば、エンジニアとも事業家とも対等に話せる土台が完成する。

04 コラム — 東の失敗から話す

私もずっと負けてきた

ピッチコンテスト・クラウドファンディング・地域ビジネス・サービス開発…
全て、起業という目標に対しては失敗だった。
入社後も毎年事業計画書を書いて、まだ勝ちきれていない。

起業とは、全責任を負い、自分のエゴで最大限利他的に、

これを突きつけられ続けて、アイデアとプロダクトを捨て続けてきた。

起業というのは、この「負け」を背負い続ける何かに、ベットしていかなければいけない。

04 コラム — 市場価値の方程式

市場価値は掛け算で決まる

市場価値Market Value
=
専門スキルSpecialized Skills
×
ドメイン知識Domain Knowledge

スキルのみ（ドメイン知識ゼロ）
高度な技術があっても現場の文脈がなければ「机上の空論」。
実運用とは程遠いプロダクトしか生まれない。

ドメインのみ（専門スキルゼロ）
現場感覚があっても突破力が不足。
「誰でも思いつく・誰でもできる」ことに収束してしまう。

「仕組み化」という抽象概念だけで戦うのは非常に危険。
今のインターン活動（多領域の現場経験）はドメイン知識の貴重な積み上げ。
そこに専門スキルを掛け算できるかどうかが、あなたの未来を分ける。

05 AI時代の戦い方

蒸気機関の時代に
馬を速く走らせても意味がない

産業変革の本質
  • AI時代には必ず産業構造が変わる
  • 「いまのやり方を最大効率化する」だけでは新しい市場で勝てない
  • どう変化するか・どのタイミングでベットするかを常に考え続けること
東が考える「最短解」
  • 組織に属して、その中で新規事業を提案する
  • 他人の会社・資産であっても全力で貢献し価値を示す
  • 自分の事業を会社の事業として成長させる動きをする
  • AIで負けた際の再挑戦もしやすい ＋ 意思決定の柔軟性を保てる

実践中の戦略： 半年ごとに評価・更新。周囲との対話で方針をアップデートし続ける。
大事なのは「辞める・続ける」ではなく、最善の判断を最速で更新し続けること。

05 AI時代の戦い方 — マネジメント再定義

「自分が働かなくても稼げる」
≠ マネジメント

×誤解

「不労所得 = マネジメント」という誤解

仕組みを作っただけでは「仕組みオーナー」でしかない。
本当のマネジメントはもっと能動的な行為である。

◎正解

マネジメントの正体

自分のプレイヤーとしての強みを組織に昇華し、
自分ができることの幅を広げて組織の再現性を強化すること。
マネジメント層は、より責任の重いこと・誰も再現できないことをし続ける必要がある。

？問い

高度な管理すら委任できる状態は、最上級ではなく一つの選択肢

自分の高度なマネジメント手法すら、誰かに役割を担ってもらい回る状態にはなれる。
しかし、市場が変化し続ける中で「何もしなくても求められる」「競合が追いつかない」状態は永続しない。
そこに対して自分が変化の起因になること、組織運営のゴールに向けてアクション・挑戦をし続けること。
それこそが重要なのでは？ — という問いを持ち続けてほしい。

過去の自分へのアドバイス — あなたへも

東が過去の自分に贈る
3つのメッセージ

1

常に3歩以上先の仮説を持ち続けること

仮説思考・ロジカルシンキングを極める。
動く前に「その先に何が起きるか」を常に3手先まで読んでいる状態を維持する。

2

転んでも継続できる最大限のリスクを取り続ける

チームで言う「リカバリー」の文化。
確実に転んでも目標だけは達成する。最大限成長してリスクに対応することを重視。
リスクを避けるのではなく、リスクを取りながら立て直す力を鍛える。

3

選択肢を選ばない（全てを喜んで受け入れる）

やりづらい人間関係・苦手なスキル・未経験のこと・無茶な要求・
釣り合わない報酬・理不尽なトラブル・自分起因でない問題…
これら全てを喜んで受け入れることが、最速の成長エンジンになる。

06 ミッション発表

取引の提案

東が提供するもの

  • 新規受講は基本行わないが、今回は特別に対応
  • AI活用・エンジニアリングだけでなく、仕事の仕方・高パフォーマンスを出すための東のロジックを全部教える
  • プロジェクトメンバーとして並走しながら育てる

提案内容

  • テーマ設定は東に一任
  • 東の教育手法を仕組み化・システム化・AI化する
  • 「東を勝たせる仕組み」をともに構築する
  • AI教育のあり方を変えるプロダクトの第一弾
進め方
まず1〜2週間でプロジェクトのゴールを設定。
その後はプロジェクトメンバーとして、常に少し先のレベルを求め続ける形で教育を行う。
ついてこれるなら、それがスタートラインになる。

なぜこのミッションか： AI活用における変化の土壌への対応は、やり方さえ正しければ再現性があり価値を生むものだと確信している。
そのロジックを実証・発展させるプロジェクトとして位置づけている。
